// The Problem

Why Not Just Use Claude & Google Docs?

You can. Most people do. And most people get mediocre output they have to rewrite anyway.

The standard approach

Open Claude. Paste context. Prompt. Get a draft. Copy to Google Docs. Edit. Realize you forgot your tone rules. Re-paste. Re-prompt. Copy again. Every. Single. Time.

This setup

Your docs and your AI live in the same workspace. The agent reads your files live: ideas, style rules, performance data and templates, then generates directly into your vault. No copy-paste. No re-explaining.

Advantages

Model flexibility: You're not locked into one provider. Cursor lets you switch between Claude, GPT, Gemini and others. The model is the engine but your context docs are the steering wheel. Swap the engine anytime. The direction stays the same.
A note on examples: This playbook uses a LinkedIn post drafting workflow as the demo. But the system works for anything marketing-related: brand guidelines, messaging frameworks, campaign briefs, content calendars, competitive research, SOPs, launch plans and execution tracking. The LinkedIn example is the demo. Your use case is whatever you need.
// The Stack

Two Tools. That's It.

No code required. Your vault is all markdown. If you can write a bullet list you can run this system.

Obsidian

Your knowledge vault. A free, offline-first personal knowledge base built on plain markdown files. No proprietary format, no lock-in.

  • Plain .md files in folders you control
  • YAML frontmatter for status & classification
  • Templates with Templater automation
  • Wiki-links between ideas, drafts and published posts
  • Plugins: Kanban, tasks and dataview queries

Cursor

An AI-powered IDE that doesn't care you're not coding. It reads and writes files. Your vault is just a folder. Open it and go.

  • Agent reads your vault files in real time
  • Skills encode your voice, rules and templates
  • Structured modes: Ask, Plan, Execute
  • Switch models per task (Claude, GPT, Gemini)
  • Creates and edits files directly, no copy-paste

Why Frontmatter Matters

Every file starts with YAML frontmatter, a metadata block the AI can read and act on. Status, category, dates. This is how you track content through the pipeline and give the AI classification context without burying it in the body.

idea-file.md

The status field moves from idea → draft → editing → published. Consistent inputs = consistent outputs.

// Context Layer

Context Before Content

AI without context produces generic output. Context docs are the difference between "meh" and "on-brand." Build them once. The AI reads them every time.

Skills ~/.cursor/skills/

Reusable instruction files that tell the AI how to do a task. Workflow steps, content pillars, post structure, tone rules and output format. The Skill is the instruction manual, and without it the AI guesses.

SKILL.md frontmatter
--- name: linkedin-draft-writer description: Write LinkedIn post drafts from idea files. Use when the user wants to create a draft or transform an idea into a post. ---

Style References linked from Skill

Performance data, hook examples, CTA patterns and voice examples. Real data so the AI knows what works, not generic advice.

// Workflow

The Three Modes

Most people skip straight to execution. That's why most AI output needs heavy rework. The discipline: research first, plan second, execute third.

🔍
Ask
Research. Read-only. Gather context.
→
📋
Plan
Structure. Questions. No edits yet.
→
⚡
Execute
Generate. Write files. Ship.
ASK

Research Before You Act

The agent explores your vault, reads files and searches but cannot edit anything. It's the briefing before the mission.

When to use: Scoping a task, understanding what exists and gathering context before decisions.
PLAN

Structure Before You Build

The agent creates a plan, asks clarifying questions and proposes the approach, still no edits. Nothing changes until you approve.

Real example: When creating this playbook, Plan mode forced two questions before writing: "Is this external-facing or internal?" and "Where should the file live?" Without it the agent would have assumed both answers.
EXECUTE

Do the Work

Context loaded. Plan approved. Scope clear. The agent reads, writes, creates files and executes.

Real example: The agent read the idea file, loaded the Skill & style reference, classified the pillar, generated the draft in template format and wrote it to Drafts/. No guessing. Every step informed by context docs.

Skipping Ask = the agent doesn't know what you have. Skipping Plan = assumptions you'll fix later. Skipping both = generic output with your name on it.

// Decomposition

Micro-Task Decomposition

AI fails on big tasks. Not because it's stupid but because large, vague instructions dilute focus. The fix: break every task into small steps with clear inputs and outputs.

StepMicro-TaskInputOutput
01Read the idea fileIdeas/Feb 19, 2026 Idea.mdRaw topic understood
02Read the SkillSKILL.mdRules & template loaded
03Read style referencestyle-reference.mdPerformance data & tone
04Classify pillarTopic & definitions"Opinion / Industry Truths"
05Generate draftAll context aboveComplete post in template
06Write to DraftsGenerated contentDrafts/Feb 19, 2026.md

Each step feeds the next. No step is ambiguous. The agent moves through them in order, not all at once.

When prompting: Give the agent ONE job at a time. "Read this file and tell me what pillar it fits" beats "read this, classify it, write a draft, pick the right hook and make it punchy."
// Verification

Step-by-Step Verification

Don't wait until the end to check if it worked. Verify after each phase.

Phase 01

After Setup

  • Vault folder structure exists
  • Frontmatter is consistent across templates
  • Templates create files correctly
  • Obsidian can find and link files
Phase 02

After Context Docs

  • Skill has correct YAML frontmatter
  • Description includes trigger terms
  • Skill body under 500 lines
  • Style ref has real performance data
Phase 03

After Generation

  • Output has correct frontmatter
  • Follows template structure
  • Tone matches style reference
  • CTA is specific and valuable
Phase 04

The Final Test

  • "Would I post this?"

If it needs more than light editing the problem isn't the prompt. Fix the context docs, not the prompt.

// Quick Start

Get Running in 30 Minutes

The condensed sequence. Follow it top to bottom.

01
Install Obsidian

obsidian.md. Free. Create a new vault.

02
Install Cursor

cursor.com. Free tier works. Open your vault folder in Cursor.

03
Create the folder structure

Ideas/, Drafts/, Templates/, Assets/Frameworks/

04
Create an Ideas template

Markdown file with YAML frontmatter (pillar, status, topic and summary).

05
Write your first Skill

~/.cursor/skills/your-skill/SKILL.md. Include tone rules, post structure and content categories.

06
Add a style reference

Link it from your Skill. Include examples of good vs. bad output and your actual voice.

07
Write a raw idea

Drop it in Ideas/. Doesn't need to be polished. That's the AI's job.

08
Ask the agent to draft it

Open Cursor. Ask it to create a draft from your idea. Watch it read, classify and generate.

09
Review the output

Check it against the verification checklist above.

10
Iterate on the Skill file

Keep refining until the output sounds like you wrote it.
